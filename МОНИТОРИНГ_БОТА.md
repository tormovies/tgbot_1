# Мониторинг бота: чтобы не погас

Есть два подхода: **cron-сторож** (проверяет раз в N минут и запускает, если процесса нет) и **systemd** (сервис с автоперезапуском и стартом при загрузке сервера).

Каталог бота на **website.com.ru** (пример): `/home/admin/domains/website.com.ru/public_html/98` — подставь своё имя папки, если другое.

---

## 1. Cron-сторож (подходит для любого сервера)

Скрипт проверяет, запущен ли процесс бота; если нет — запускает.

### 1.1. Скрипт уже в репозитории: `bot-watchdog.sh`

Положи `bot-watchdog.sh` в **корень папки бота** (рядом с `bot.php`). Тогда `BOT_DIR` в скрипте вычисляется сам из расположения файла.

При необходимости отредактируй вручную путь в скрипте (если скрипт лежит не в корне бота):

```bash
nano /home/admin/domains/website.com.ru/public_html/98/bot-watchdog.sh
```

Сделай скрипт исполняемым:

```bash
chmod +x /home/admin/domains/website.com.ru/public_html/98/bot-watchdog.sh
```

### 1.2. Запускать по cron каждые 5 минут

```bash
crontab -e
```

Добавь строку (путь — полный к `bot-watchdog.sh` в папке бота):

```
*/5 * * * * /home/admin/domains/website.com.ru/public_html/98/bot-watchdog.sh
```

Сохрани и выйди. Так каждые 5 минут будет проверка: если бота нет — он запустится.

Проверить, что задание добавилось:

```bash
crontab -l
```

## 2. Systemd (автоперезапуск и старт при загрузке, обычно VPS/свой сервер)

Если есть root и systemd, можно сделать сервис: при падении бота он перезапустится, при перезагрузке сервера — запустится сам.

### 2.1. Создать unit-файл

Имя файла сервиса произвольное, ниже пример `bot98.service`:

```bash
sudo nano /etc/systemd/system/bot98.service
```

Содержимое (путь и User замени на свои):

```ini
[Unit]
Description=Telegram dream bot (website.com.ru /98)
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/home/admin/domains/website.com.ru/public_html/98
ExecStart=/usr/bin/php bot.php
Restart=always
RestartSec=10
StandardOutput=append:/home/admin/domains/website.com.ru/public_html/98/data/bot.log
StandardError=append:/home/admin/domains/website.com.ru/public_html/98/data/bot.log

[Install]
WantedBy=multi-user.target
```

Прокси к Telegram задаётся в **`.proxy.env`** в корне бота — его читает сам **`bot.php`**. Для unit systemd при необходимости добавь строки вида `Environment=HTTPS_PROXY=http://IP:8888` в секцию `[Service]`. Замени **User=** и **WorkingDirectory** на свои, если нужно. Сохрани.

### 2.2. Включить и запустить

```bash
sudo systemctl daemon-reload
sudo systemctl enable bot98
sudo systemctl start bot98
sudo systemctl status bot98
```

Дальше бот управляется так:

- статус: `sudo systemctl status bot98`
- перезапуск: `sudo systemctl restart bot98`
- остановка: `sudo systemctl stop bot98`

После перезагрузки сервера сервис поднимется сам.

## 3. Быстрая ручная проверка

Проверить, что процесс есть:

```bash
pgrep -af bot.php
```

Последние строки лога:

```bash
tail -20 /home/admin/domains/website.com.ru/public_html/98/data/bot.log
```

Если поставил cron-сторож — при следующей «смерти» бота он появится снова в течение 5 минут, и в лог добавится строка вида: `Bot was down, started.`
